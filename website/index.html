Temporal memory for AI agents.

Quipu is a local-first memory system that keeps raw evidence, extracts temporal facts with provenance, and retrieves structured context packets. Facts expire. Deletion propagates.

GitHub → Read the spec →
What it is

Most agent memory is a vector index: loose text fragments with no sense of time, no link to source, and no way to forget. Quipu treats memory as a graph of evidence, derived facts, and procedures — each annotated with scope, time, and lineage. When evidence changes, everything downstream updates.

Protocol

Remember first. Retrieve when context matters.

The public API is JSON-RPC. SDKs validate inputs and forward to the runtime; they do not reimplement extraction, retrieval, or temporal logic.

Architecture

One local runtime owns the semantics.

The Zig runtime is the source of truth. SDKs and tools submit requests via JSON-RPC. The runtime handles evidence storage, fact extraction, temporal queries, and propagation forgetting.

Status

Prototype. Not a hosted service.

Implemented

  • Zig core runtime and CLI protocol dispatch.
  • JSON-RPC methods: health, remember, retrieve, search, inspect, forget, feedback.
  • In-memory adapter with graph records, streams, search, and invariants.
  • Optional LatticeDB adapter for durable local storage.
  • TypeScript, Python, and MCP stdio surfaces.
  • Conformance fixtures and smoke evals.

Planned

  • Socket/HTTP daemon mode.
  • Provider embeddings, BM25, reranking.
  • LLM-backed extraction and consolidation workers.
  • Richer MCP integrations.
  • Publication-quality experiments.
